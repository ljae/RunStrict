# Rephrase & Confirm Protocol

## Rule (ALWAYS ACTIVE — NO EXCEPTIONS)

Whenever the user writes a command or request for you to act on:

1. **Rephrase** it — rewrite with clearer, more precise, professional wording that captures the full intent
2. **Show** the rephrased version to the user
3. **Wait** for explicit confirmation (e.g. "yes", "correct", "proceed") before doing anything

## Format

Present the rephrased command like this:

> **Rephrased:** [your improved version of the command]
>
> Proceed?

## What counts as a "command"

- Any request to implement, create, edit, delete, refactor, or run something
- Any instruction to analyze, investigate, or research something
- Any multi-step task

## What does NOT require rephrase+confirm

- Simple follow-up questions or clarifications
- Answering a factual/informational question with no action taken
- The user is responding to YOUR question

## After confirmation

Once the user confirms, proceed normally with full execution. Do not ask again mid-task.
